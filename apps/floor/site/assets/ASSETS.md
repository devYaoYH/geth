# The floor's asset kit — extension grammar

Every sprite here is generated by `tools/gen_assets.py`. Do not hand-edit
the SVGs: change the generator, rerun it, commit both. Determinism is the
contract that keeps the floor extensible — a new node drawn by a different
agent two milestones from now must sit in the same projection and palette
as everything drawn today.

## Projection

2:1 dimetric ("isometric" in the pixel-art sense). Inside a sprite, local
grid coordinates map to screen as:

    screen_x = anchor_x + (x - y) * 2
    screen_y = anchor_y - (x + y) - z * 2

The renderer (`iso.js`) uses the same projection for floor tiles, with one
tile = 24 grid units, so sprites and floor lock together at any scale.

## Sprite contracts

| kind    | viewBox     | anchor (must sit at)          |
|---------|-------------|-------------------------------|
| decor   | 0 0 128 128 | (64, 104) = room's south corner |
| worker  | 0 0 32 40   | (16, 38) = feet               |
| prop    | 0 0 64 64   | (32, 46) = base center        |

Light comes from the upper-left: top faces at 1.12x base color, southwest
faces 0.88x, southeast faces 0.68x (the `Sprite.box` helper does this).
Every grounded sprite carries a soft `#000 @ 0.13` ellipse shadow.

Stay inside it. New hues make a room look imported instead of built.

## Room archetypes shipped

worldgate (the internet) · gatehouse (ingress proxy) · identity · stamp
(auth shim) · dock (egress) · reactor (LLM gateway) · vault (database) ·
archive (git) · catalog (registry) · console (dashboard) · periscope
(docker status) · bell (dispatch trigger) · workbench (dev agent) ·
frontdesk (assistant) · pad (ephemeral tenant) · calendar · antenna
(feeds) · pinboard (notes) · switchboard (chat) · arcade (game) · radar
(search) · mailroom (mail/drive bridge) · drafting (this floor itself) ·
workshop (the generic fallback — every unknown service gets this, so a
new node is never invisible).

## Workers (data movement)

courier (parcel = generic payload / git objects) · spark (bolt = LLM
inference) · scroll (document = issues, PRs, config) · inspector
(clipboard = health checks, audits). The renderer picks the worker from
the event kind (`iso.js` WORKER_FOR); a new event kind falls back to
courier.

## Adding a room kind for a new service

1. Most new services need nothing: the registry manifest gives them a
   generic workshop room automatically, with ingress + `needs` edges.
2. Bespoke room: add `def deco_<name>(s)` in `tools/gen_assets.py`
   (compose with `s.box()` iso boxes + small `s.raw()` overlays), rerun
   the generator.
3. Map services to it: a keyword in `ARCHETYPE_RULES` (app.py) for
   manifest-described apps, or a `CORE_ROOMS` entry for skeleton
   services. Fixed plumbing edges go in `CORE_EDGES`.
4. New data-flow kind: add a worker in `WORKERS` (chest color + carried
   item on the right hip) and a `WORKER_FOR` entry in iso.js.

That is the whole extension surface: one sprite function, one rule line.
